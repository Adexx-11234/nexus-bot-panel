### Overview

Nexus Bot features an **enterprise-grade permission system** with:
- âœ… **Centralized Control** - All permissions in one place
- âœ… **Cached Validation** - 30-second cache for performance
- âœ… **Flexible Configuration** - 8 permission types
- âœ… **Automatic Enforcement** - No manual checks in plugins

### Permission Types

#### User Permissions

| Permission | Who Can Use | Use Case | Example |
|------------|-------------|----------|---------|
| `ownerOnly` | Bot owner only | System commands | `eval`, `restart` |
| `vipRequired` | VIP users (strict) | Premium features | `bugmenu` |
| `ownerOrVip` | Owner OR VIP | VIP management | `vipadd` |
| `ownerAndVip` | Owner with VIP | Ultra-restricted | `vipadmin` |
| `defaultVipOnly` | Default VIP/Owner | VIP admin | VIP panel |
| `adminRequired` | Group admins | Group management | `kick`, `promote` |

#### Bot & Context Permissions

| Permission | Description | Example |
|------------|-------------|---------|
| `botAdminRequired` | Bot must be admin | `kick`, `delete` |
| `groupOnly` | Groups only | Group commands |
| `privateOnly` | Private chats only | Personal settings |

### Permission Examples

#### 1ï¸âƒ£ Owner Only Command

#### 2ï¸âƒ£ VIP Command (Strict)

#### 3ï¸âƒ£ Owner OR VIP (Flexible)

#### 4ï¸âƒ£ Owner AND VIP (Ultra-Strict) â­

#### 5ï¸âƒ£ Group Admin Command

```javascript
  adminRequired: true,
  botAdminRequired: true,
  groupOnly: true

// âœ… Group admins
// âœ… Bot owner (auto-pass)
// âŒ Regular users
```

#### 6ï¸âƒ£ Anti-Plugin (Inverted Logic)
```javascript

  adminRequired: true,      // Non-admins processed
  botAdminRequired: true,   // Bot needs admin
  groupOnly: true
}
// Processes: Regular users
// Bypasses: Admins, VIPs, Owner
```

### âš ï¸ Important Rules

1. **Use ONE primary permission** - Don't combine `ownerOnly` with `vipRequired`
2. **Context permissions are additive** - Can combine with user permissions
3. **Bot permissions are requirements** - `botAdminRequired` is separate
4. **Anti-plugins use inverted logic** - `adminRequired` = non-admins get processed

**âŒ WRONG:**
```javascript
permissions: {
  ownerOnly: true,
  vipRequired: true,        // Conflict!
  defaultVipOnly: true      // Conflict!
}
```

**âœ… CORRECT:**
```javascript
permissions: {
  ownerAndVip: true,        // One primary permission
  privateOnly: true         // Context restriction
}
```

### PostgreSQL Tables

| Table | Purpose | Key Fields |
|-------|---------|------------|
| `users` | User accounts | telegram_id, phone, first_name |
| `whatsapp_users` | WA settings | jid, prefix, bot_mode |
| `groups` | Group settings | jid, anti_features |
| `messages` | Message history | Auto-cleanup at 10k rows |
| `warnings` | User warnings | group_jid, user_jid, count |
| `violations` | Anti-feature logs | type, action, timestamp |
| `vip_owned_users` | VIP ownership | vip_id, owned_user_id |
| `vip_activity_log` | VIP actions | action_type, details |
